# 什么是 MCP？

MCP（Model Context Protocol，模型上下文协议）是一个通过专用服务器处理工具和资源调用的简单协议。

**模型上下文协议 (MCP) 是由 Anthropic 推出的一种开放标准，旨在简化大型语言模型 (LLM) 与外部工具及数据源的集成。你可以将 MCP 想象成 AI 应用的“USB-C 接口”：正如 USB-C 标准化了设备之间的连接一样，MCP 为 AI 模型访问各种数据 and 工具提供了一种标准化的方式。**

模型上下文协议 (MCP) 是一种开放标准，使大型语言模型能够通过标准化接口与外部工具、数据库和 API 进行动态交互。

模型上下文协议 (MCP) 是由 Anthropic 开发的一项开放标准，旨在实现 AI 模型与外部工具之间简单且标准化的集成。它充当通用连接器，允许大型语言模型 (LLM) 与 API、数据库和商业应用进行动态交互。

最初是为了提高 Claude 与外部系统交互的能力而构建的，Anthropic 决定在 [2024 年初开源 MCP](https://www.anthropic.com/news/model-context-protocol)，以鼓励全行业的采用。通过公开 MCP，他们的目标是创建一个 AI 到工具通信的标准框架，减少对专有集成的依赖，并实现 AI 应用之间更强的模块化和互操作性。

> _这可以理解为一个向 AI 提供特定上下文的模块。_

## 构建 MCP 服务器的核心原则

在教程中，作者演示了如何构建一个可以与 Claude Desktop 集成的 MCP 服务器，以实现以下功能：

1. **获取 PR 详情和更改的文件**：通过 GitHub API，服务器可以获取指定 PR 的详细信息，包括更改的文件列表。
2. **分析代码更改**：使用 Claude Desktop 应用程序，服务器可以分析代码更改的内容并生成评审摘要和建议。
3. **将评审结果保存到 Notion**：生成的评审摘要和建议被保存到 Notion，供团队成员查看和跟踪。
4. **整个过程通过 MCP 实现了标准化通信，使得服务器与 Claude Desktop 之间的交互更加模块化和可扩展。**

总的来说，就是先搭建环境（Python），然后编写相关脚本来实现功能（Python），最后进行配置和调试，直接使用。

Anthropic 本身有许多第三方平台提供了大量的 MCP 服务器，你可以在这个 [链接](https://github.com/modelcontextprotocol/servers) 中查看。

**需要注意的一个重要点：Claude 官方的** **[开发者使用 MCP 服务器文档](https://modelcontextprotocol.io/quickstart/server#windows) 包含了初始配置文件所需的代码格式。**

最后，写入成功了。PS：Notion 需要创建一个集成（integration）并创建一个页面才能正确写入。
